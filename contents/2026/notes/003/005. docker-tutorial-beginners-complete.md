## Docker Basics

### What is Docker?

**Docker** is:
- **Containerization Platform**: Package applications with dependencies
- **Lightweight Virtualization**: More efficient than VMs
- **Portable**: Run anywhere (dev, test, prod)
- **Isolated**: Isolated environments for applications
- **Scalable**: Easy to scale applications

### Why Docker?

**Benefits:**
1. **Consistency**: Same environment everywhere
2. **Isolation**: Applications don't interfere
3. **Portability**: Run on any Docker host
4. **Efficiency**: Less overhead than VMs
5. **Speed**: Fast startup and deployment
6. **Scalability**: Easy horizontal scaling

### Docker vs Virtual Machines

| Aspect | Docker | Virtual Machine |
|--------|--------|-----------------|
| **OS** | Shares host OS | Full OS per VM |
| **Size** | MBs | GBs |
| **Startup** | Seconds | Minutes |
| **Performance** | Near native | Overhead |
| **Isolation** | Process level | Hardware level |
| **Resource Usage** | Low | High |

### Docker Architecture

```
┌─────────────────────────────────────┐
│         Docker Client               │
│  (docker CLI, Docker Desktop)        │
└──────────────┬──────────────────────┘
               │
               ↓
┌─────────────────────────────────────┐
│         Docker Daemon               │
│  (dockerd - manages containers)     │
└──────────────┬──────────────────────┘
               │
    ┌──────────┼──────────┐
    ↓          ↓          ↓
┌────────┐ ┌────────┐ ┌────────┐
│Container│ │Container│ │Container│
└────────┘ └────────┘ └────────┘
```

**Components:**
- **Docker Client**: CLI tool (docker command)
- **Docker Daemon**: Background service (dockerd)
- **Docker Images**: Templates for containers
- **Docker Containers**: Running instances of images
- **Docker Registry**: Store and share images (Docker Hub)

### Key Concepts

**1. Image**
- **Template**: Read-only template for containers
- **Layers**: Built from multiple layers
- **Immutable**: Images don't change
- **Reusable**: Can create multiple containers from one image

**2. Container**
- **Instance**: Running instance of an image
- **Isolated**: Isolated from host and other containers
- **Ephemeral**: Can be created/destroyed easily
- **Portable**: Can run on any Docker host

**3. Dockerfile**
- **Blueprint**: Instructions to build an image
- **Commands**: Series of commands
- **Layers**: Each command creates a layer
- **Build**: Used to build images

## Installation & Setup

### Installation on Linux

**Ubuntu/Debian:**
```bash
# Update package index
sudo apt-get update

# Install prerequisites
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# Add Docker's official GPG key
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Set up repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Start Docker
sudo systemctl start docker
sudo systemctl enable docker

# Verify installation
sudo docker run hello-world
```

**CentOS/RHEL:**
```bash
# Install prerequisites
sudo yum install -y yum-utils

# Add Docker repository
sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

# Install Docker Engine
sudo yum install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Start Docker
sudo systemctl start docker
sudo systemctl enable docker

# Verify installation
sudo docker run hello-world
```

## Docker Images

### What is a Docker Image?

**Docker Image:**
- **Read-only Template**: Template for containers
- **Layered**: Built from multiple layers
- **Immutable**: Images don't change
- **Reusable**: Create multiple containers

### Image Layers

**Layer Structure:**
```
┌─────────────────────┐
│   Application       │  ← Top layer (writable in container)
├─────────────────────┤
│   Dependencies      │  ← Layer 3
├─────────────────────┤
│   Runtime           │  ← Layer 2
├─────────────────────┤
│   Base OS           │  ← Layer 1 (base image)
└─────────────────────┘
```

**Benefits:**
- **Reusability**: Share common layers
- **Efficiency**: Only download changed layers
- **Speed**: Faster builds and pulls

### Working with Images

**Pull Image:**
```bash
# Pull latest version
docker pull nginx

# Pull specific version
docker pull nginx:1.21

# Pull from specific registry
docker pull registry.example.com/image:tag
```

**List Images:**
```bash
# List all images
docker images

# List with filters
docker images nginx
docker images --filter "dangling=true"

# List image IDs only
docker images -q
```

**Inspect Image:**
```bash
# Inspect image details
docker inspect nginx

# Inspect specific field
docker inspect -f '{{.Config.Cmd}}' nginx
```

**Remove Image:**
```bash
# Remove image
docker rmi nginx

# Remove by ID
docker rmi abc123def456

# Force remove
docker rmi -f nginx

# Remove all images
docker rmi $(docker images -q)

# Remove dangling images
docker image prune
```

**Search Images:**
```bash
# Search Docker Hub
docker search nginx

# Search with filters
docker search --limit 5 nginx
```

## Docker Containers

### What is a Docker Container?

**Docker Container:**
- **Running Instance**: Running instance of an image
- **Isolated**: Isolated from host and other containers
- **Ephemeral**: Can be created/destroyed easily
- **Portable**: Can run on any Docker host

### Container Lifecycle

```
Create → Start → Running → Stop → Remove
         ↑                    ↓
         └────── Restart ──────┘
```

### Working with Containers

**Run Container:**
```bash
# Run container in foreground
docker run nginx

# Run container in background (detached)
docker run -d nginx

# Run with name
docker run --name my-nginx nginx

# Run with port mapping
docker run -p 8080:80 nginx

# Run with environment variables
docker run -e VAR=value nginx

# Run with volume
docker run -v /host/path:/container/path nginx

# Run with interactive terminal
docker run -it ubuntu /bin/bash

# Run and remove on exit
docker run --rm nginx
```

**List Containers:**
```bash
# List running containers
docker ps

# List all containers
docker ps -a

# List with filters
docker ps --filter "status=running"
docker ps --filter "name=nginx"

# List container IDs only
docker ps -q
```

**Start/Stop Containers:**
```bash
# Start container
docker start container-name

# Stop container
docker stop container-name

# Restart container
docker restart container-name

# Pause container
docker pause container-name

# Unpause container
docker unpause container-name

# Kill container (force stop)
docker kill container-name
```

**Execute Commands in Container:**
```bash
# Execute command in running container
docker exec container-name ls

# Execute interactive command
docker exec -it container-name /bin/bash

# Execute as specific user
docker exec -u root container-name whoami
```

**View Container Logs:**
```bash
# View logs
docker logs container-name

# Follow logs
docker logs -f container-name

# Last N lines
docker logs --tail 100 container-name

# Since timestamp
docker logs --since 2024-01-01T00:00:00 container-name
```

**Inspect Container:**
```bash
# Inspect container
docker inspect container-name

# Inspect specific field
docker inspect -f '{{.NetworkSettings.IPAddress}}' container-name
```

**Remove Container:**
```bash
# Remove stopped container
docker rm container-name

# Force remove running container
docker rm -f container-name

# Remove all stopped containers
docker container prune

# Remove all containers
docker rm $(docker ps -aq)
```

**Copy Files:**
```bash
# Copy from container to host
docker cp container-name:/path/file.txt /host/path/

# Copy from host to container
docker cp /host/path/file.txt container-name:/path/
```

## Dockerfile

### What is a Dockerfile?

**Dockerfile:**
- **Blueprint**: Instructions to build an image
- **Text File**: Plain text file
- **Commands**: Series of commands
- **Build**: Used with `docker build`

### Dockerfile Instructions

**FROM:**
```dockerfile
# Base image
FROM ubuntu:20.04

# Use specific tag
FROM node:16-alpine

# Use digest for reproducibility
FROM nginx@sha256:abc123...
```

**RUN:**
```dockerfile
# Execute command
RUN apt-get update
RUN apt-get install -y nginx

# Chain commands
RUN apt-get update && \
    apt-get install -y nginx && \
    rm -rf /var/lib/apt/lists/*

# Multi-line for readability
RUN apt-get update \
    && apt-get install -y nginx \
    && rm -rf /var/lib/apt/lists/*
```

**COPY:**
```dockerfile
# Copy file
COPY app.py /app/

# Copy directory
COPY src/ /app/src/

# Copy with wildcard
COPY *.txt /app/

# Copy with specific ownership
COPY --chown=user:group file.txt /app/
```

**ADD:**
```dockerfile
# Similar to COPY but can:
# - Download from URL
# - Extract tar files
ADD https://example.com/file.tar.gz /app/
ADD archive.tar.gz /app/
```

**WORKDIR:**
```dockerfile
# Set working directory
WORKDIR /app

# Subsequent commands run in this directory
RUN pwd  # Output: /app
```

**ENV:**
```dockerfile
# Set environment variable
ENV NODE_ENV=production
ENV PORT=8080

# Multiple variables
ENV NODE_ENV=production \
    PORT=8080 \
    API_KEY=secret
```

**ARG:**
```dockerfile
# Build-time argument
ARG VERSION=latest
ARG BUILD_DATE

# Use in build
RUN echo "Building version $VERSION"
```

**EXPOSE:**
```dockerfile
# Document port (doesn't publish)
EXPOSE 80
EXPOSE 8080

# Multiple ports
EXPOSE 80 443 8080
```

**CMD:**
```dockerfile
# Default command (can be overridden)
CMD ["nginx", "-g", "daemon off;"]

# Shell form
CMD nginx -g "daemon off;"
```

**ENTRYPOINT:**
```dockerfile
# Entry point (not easily overridden)
ENTRYPOINT ["nginx", "-g", "daemon off;"]

# Can be combined with CMD
ENTRYPOINT ["docker-entrypoint.sh"]
CMD ["nginx"]
```

**USER:**
```dockerfile
# Switch user
USER nginx

# Switch user and group
USER nginx:nginx
```

**VOLUME:**
```dockerfile
# Create mount point
VOLUME ["/data"]

# Multiple volumes
VOLUME ["/data", "/logs"]
```

**LABEL:**
```dockerfile
# Add metadata
LABEL maintainer="your@email.com"
LABEL version="1.0"
LABEL description="My application"
```

### Complete Dockerfile Example

**Node.js Application:**
```dockerfile
# Base image
FROM node:16-alpine

# Set working directory
WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci --only=production

# Copy application code
COPY . .

# Create non-root user
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001

# Change ownership
RUN chown -R nodejs:nodejs /app

# Switch to non-root user
USER nodejs

# Expose port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=3s \
  CMD node healthcheck.js

# Start application
CMD ["node", "server.js"]
```

**Python Application:**
```dockerfile
# Base image
FROM python:3.9-slim

# Set working directory
WORKDIR /app

# Set environment variables
ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1

# Install system dependencies
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
    gcc && \
    rm -rf /var/lib/apt/lists/*

# Copy requirements
COPY requirements.txt .

# Install Python dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Copy application
COPY . .

# Expose port
EXPOSE 8000

# Run application
CMD ["gunicorn", "--bind", "0.0.0.0:8000", "app:app"]
```

**Java Application:**
```dockerfile
# Build stage
FROM maven:3.8-openjdk-17 AS builder
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# Runtime stage
FROM openjdk:17-jre-slim
WORKDIR /app
COPY --from=builder /app/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### Building Images

**Build Image:**
```bash
# Build from Dockerfile in current directory
docker build .

# Build with tag
docker build -t myapp:1.0 .

# Build with tag and Dockerfile path
docker build -t myapp:1.0 -f Dockerfile.prod .

# Build with build arguments
docker build --build-arg VERSION=1.0 -t myapp:1.0 .

# Build without cache
docker build --no-cache -t myapp:1.0 .

# Build with target (multi-stage)
docker build --target builder -t myapp:builder .
```

**Build Context:**
```bash
# Build from specific directory
docker build -t myapp:1.0 ./app

# Build from URL
docker build https://github.com/user/repo.git

# Build from tar archive
docker build - < archive.tar
```

## Docker Compose

### What is Docker Compose?

**Docker Compose:**
- **Multi-Container**: Define and run multi-container applications
- **YAML File**: Define services in docker-compose.yml
- **Orchestration**: Manage multiple containers
- **Development**: Great for development environments

### docker-compose.yml

**Basic Example:**
```yaml
version: '3.8'

services:
  web:
    image: nginx
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html

  db:
    image: postgres:13
    environment:
      POSTGRES_DB: mydb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:
```

**Advanced Example:**
```yaml
version: '3.8'

services:
  web:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgres://user:pass@db:5432/mydb
    depends_on:
      - db
      - redis
    networks:
      - app-network
    volumes:
      - ./app:/app
      - /app/node_modules
    restart: unless-stopped

  db:
    image: postgres:13
    environment:
      POSTGRES_DB: mydb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:6-alpine
    ports:
      - "6379:6379"
    networks:
      - app-network
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - web
    networks:
      - app-network
    restart: unless-stopped

volumes:
  db-data:

networks:
  app-network:
    driver: bridge
```

**3. None Network:**


### Network Commands


## Best Practices

### 1. Use Multi-Stage Builds

```dockerfile
# Build stage
FROM node:16 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Production stage
FROM node:16-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY package*.json ./
CMD ["node", "dist/index.js"]
```

### 2. Minimize Layers

```dockerfile
# Bad: Multiple layers
RUN apt-get update
RUN apt-get install -y nginx
RUN rm -rf /var/lib/apt/lists/*

# Good: Single layer
RUN apt-get update && \
    apt-get install -y nginx && \
    rm -rf /var/lib/apt/lists/*
```

### 3. Use .dockerignore

**.dockerignore:**
```
node_modules
.git
.env
*.log
.DS_Store
```

### 4. Don't Run as Root

```dockerfile
# Create user
RUN addgroup -g 1001 -S appuser && \
    adduser -S appuser -u 1001

# Switch to user
USER appuser
```

### 5. Use Specific Tags

```dockerfile
# Bad
FROM node:latest

# Good
FROM node:16-alpine
```

### 6. Health Checks

```dockerfile
HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost:3000/health || exit 1
```

### 7. Optimize Caching

```dockerfile
# Copy dependencies first (changes less frequently)
COPY package*.json ./
RUN npm install

# Copy code last (changes more frequently)
COPY . .
```

## Troubleshooting

### Common Issues

**1. Container Won't Start:**
```bash
# Check logs
docker logs container-name

# Check container status
docker ps -a

# Inspect container
docker inspect container-name
```

**2. Port Already in Use:**

## Interview Questions & Answers

### Q1: What is Docker?

**Answer:**
- **Containerization Platform**: Package applications with dependencies
- **Lightweight**: More efficient than VMs
- **Portable**: Run anywhere
- **Isolated**: Isolated environments
- **Scalable**: Easy to scale

### Q2: What is the difference between Docker and VM?

**Answer:**
- **OS**: Docker shares host OS, VM has full OS
- **Size**: Docker is MBs, VM is GBs
- **Startup**: Docker is seconds, VM is minutes
- **Performance**: Docker is near native, VM has overhead
- **Isolation**: Docker is process level, VM is hardware level

### Q3: What is a Docker image?

**Answer:**
- **Template**: Read-only template for containers
- **Layered**: Built from multiple layers
- **Immutable**: Images don't change
- **Reusable**: Create multiple containers

### Q4: What is a Docker container?

**Answer:**
- **Instance**: Running instance of an image
- **Isolated**: Isolated from host and other containers
- **Ephemeral**: Can be created/destroyed easily
- **Portable**: Can run on any Docker host

### Q5: What is Dockerfile?

**Answer:**
- **Blueprint**: Instructions to build an image
- **Text File**: Plain text file
- **Commands**: Series of commands
- **Build**: Used with `docker build`

### Q6: What is Docker Compose?

**Answer:**
- **Multi-Container**: Define and run multi-container applications
- **YAML File**: Define services in docker-compose.yml
- **Orchestration**: Manage multiple containers
- **Development**: Great for development environments

### Q7: What are Docker volumes?

**Answer:**
- **Persistent Storage**: Persistent data storage
- **Types**: Named volumes, bind mounts, anonymous volumes
- **Data Persistence**: Data survives container removal
- **Sharing**: Share data between containers

### Q8: What is Docker networking?

**Answer:**
- **Container Communication**: Connect containers
- **Network Types**: Bridge, host, none, overlay
- **Isolation**: Network isolation between containers
- **Port Mapping**: Map container ports to host

### Q9: What are Docker best practices?

**Answer:**
1. **Multi-stage builds**: Reduce image size
2. **Minimize layers**: Combine RUN commands
3. **Use .dockerignore**: Exclude unnecessary files
4. **Don't run as root**: Use non-root user
5. **Specific tags**: Use specific image tags
6. **Health checks**: Add health checks
7. **Optimize caching**: Order commands by change frequency

### Q10: How do you debug Docker containers?

**Answer:**
- **Logs**: `docker logs container-name`
- **Inspect**: `docker inspect container-name`
- **Exec**: `docker exec -it container-name /bin/bash`
- **Events**: `docker events`
- **Stats**: `docker stats`
