### 2.3 Code Documentation

**In-Code Documentation**:
```java
class CodeDocumentation {
    
    /**
     * Code Documentation Levels:
     */
    
    public void documentCode() {
        // 1. Self-Documenting Code
        selfDocumentingCode();
        
        // 2. Comments for Why, not What
        meaningfulComments();
        
        // 3. JavaDoc for Public APIs
        javadocComments();
    }
    
    private void selfDocumentingCode() {
        // ❌ Bad: Unclear names
        String bad = "public void proc(Data d) { ... }";
        
        // ✅ Good: Clear names
        String good = "public void processOrder(Order order) { ... }";
    }
    
    private void meaningfulComments() {
        // ❌ Bad: Comments what code does
        String bad = """
            // Increment counter
            counter++;
            """;
        
        // ✅ Good: Comments why
        String good = """
            // Increment retry counter to track failed attempts
            // After 3 retries, we'll fall back to default payment method
            retryCounter++;
            """;
    }
    
    private void javadocComments() {
        String example = """
            /**
             * Processes a payment for the given order.
             * 
             * @param order The order to process payment for
             * @param paymentMethod The payment method to use
             * @return PaymentResult containing transaction ID and status
             * @throws PaymentException if payment processing fails
             * 
             * @implNote This method handles retries automatically.
             *           After 3 failed attempts, it falls back to
             *           the default payment method.
             */
            public PaymentResult processPayment(Order order, PaymentMethod paymentMethod) {
                // Implementation
            }
            """;
    }
}
```

## Summary: Part 4

### Key Takeaways

1. **Knowledge Types**: Explicit, Tacit, Procedural - each needs different transfer methods
2. **Documentation**: Living, audience-focused, with examples
3. **Teaching Methods**: Pair programming, tech talks, workshops, storytelling
4. **Culture**: Build knowledge sharing into team culture
5. **Structured Transfer**: Use frameworks for systematic knowledge transfer

### Best Practices

- **Document Explicit Knowledge**: Write it down
- **Transfer Tacit Knowledge**: Pair, mentor, discuss
- **Verify Understanding**: Ask, practice, apply
- **Maintain Knowledge**: Keep docs updated, regular reviews
- **Build Culture**: Make sharing normal and valued
